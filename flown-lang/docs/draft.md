## 型システム

データ型と所有型

- データ型: 数値型など、不変でリソースを持たない値のための型
- 参照型: データ型と同様に不変でリソースを持たない、ただし内部に他の参照を持つことができて、エスケープしない
- 所有型: 配列など、通常は可変で、通常は破棄時の処理が必要な値のための型

組み込みのデータ型

- 数値型 i32, usize, f64, c8 など
- unit, bool

組み込みの参照型

- &T
- &mut T
- &T | null

struct

- いくつかのフィールドからなる型
- データ型か所有型かは明示的に指定する
- データ型は所有型のフィールドを持てない

```rust
struct Point(f64, f64) // データ型

ref struct Item(array: &Array[T], index: usize) // 参照型

owned struct String(...) // 所有型
```

enum

- いくつかのバリアントからなる、判別共用体

コンテナ

- 配列 `Array[T]`
    - 可変な動的配列
    - 要素の型 T はデータ型に限られる
- マップ Map[K, T]
    - 可変なハッシュマップ
    - キーと要素はデータ型に限られる

文字列

- 短い文字列 string はデータ型?
- 長い文字列 BigString は所有型
- 文字列リテラルの型は文脈に応じてどっちかになる

関数

- 関数ポインタの型 fn(param) -> result
- 状態を持たない関数のトレイト Fn(param) -> result
- データへの参照を持つ関数のトレイト RefFn(param) -> result
- 可変な状態を持つ関数のトレイト OwnedFn(param) -> result

特殊な型

- unknown
- never

## メモリ管理

- 所有型はデストラクタを持つ
    - 所有型のオブジェクトは確保されたら必ず1回破棄される。破棄後には使用されない
- 一定の制約をかけることで、所有型が破棄されるのをスコープの末尾に限定する
    - 途中で破棄されることはないし、破棄されずにスコープを抜けることもない、というようにしたい
    - (GCなしでメモリ管理しつつ、高速かつ安全なメモリ管理を保証し、Rustよりシンプルなルールにしたい)
- ローカル変数がスコープを抜ける際に、その型のデストラクタを起動して、リソースを破棄する (RAII)
- オブジェクトは move できない
- 所有型のオブジェクトの動的な確保・解放に制限をかける
    - 配列やマップの要素として所有型は認めない
    - 所有型からなる配列は作れない
